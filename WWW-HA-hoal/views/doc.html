Inspiration and Idea

Ballon is a tourism agency which tries to give its customers best trips of their lives. Only problem is that it had no online presence this website enables them to give customer a platform where they can know more about the agency and know what kind of trips they offer. Not only that, customers and potential customers can create accounts and comment on the trips itself to express their likings and disliking.

Website Stack

This website is built with the Node js webstack. Because I used the node js server framework express.js to develop backend endpoints and used Nunjucks as the template engine to write the views. Used JWT token for authentication purposes and SQLITE as the database of choice.

Development lifecycle

I developed the website on an iterative model. I developed first how one feature will look, created the front end side, then created the back end endpoint and database service implementation, and then tested how it would be integrated with other functions. I developed a search function, a registration function and a comment function while I did the rest.

Backend

The backend development included developing the appropriate database service for a database entry, creating a route entry for this entry, and registering this route file for use in the server.js file. For example, to create a comment on places to visit, we first need to create a function in the dbconfig.js file that creates that particular table in the database to store comment data. After creating this script, the commentsService.js file was created which hosts all functions that interact directly with the database to perform CRUD on that particular table. After creating this file, I created the commentsRoutes.js file that hosts the backend endpoints, specifically with regards to the comment entity in the application.

Frontend

For frontend development i use Nunjucks templating engine which has been recommended by requirement document. Nunjucks is different than other templating engines in such that it does not need templates file to be of certain kind. It works on html files. To develop a frontend page the typical process was to identify which parts of the view are conditional and which parts are to be looped and the writing nunjucks logic accordingly. For example, on base.html page, there had to put a conditional logic to show logout and profile buttons if user is signed in otherwise show login and signup links. There was not much use of Javascript in the frontend as this is server rendered app which means most of the work is being done by nunjucks at the backend of the application to spit out the html and css for the website.

Application flow

Application flow can be divided for the three different user types

Visitor flow

A normal visitor can only see About My Pages and landing pages with team pages. He can't comment on landing pages, but he can read other people's comments

Logged In user flow

A user who has created an account can do above things as well as can update his account, with the perk to add his own comments publically.

Admin flow

An admin is special user who not only can do above stuff, he can manage all the data on the website. He can delete abusive comments, users also manage and update destinations showed on the website. Sign In- Username:admin-Password:123

Future Enhancements

  1. Users can like/dislike a comment or destination
  2. More control for user accounts on profile page
  3. Google maps integration for destinations

Lessons learnt

    It is very important to work out domain objects and data models before you start coding. It was a small website that I developed using an iterative approach, but it showed me the importance of solidifying the models early in the development lifecycle. When your domain entities are clear, you can get the job done neatly, too.
